# Wiki、图谱与版本

## 引言

本页说明入库完成后如何检查长期知识资产。资料是证据，Wiki 页面、Graph View、Knowledge Check 和版本记录是知识库的工作结果。

## 步骤 1：查看 Wiki 页面

进入 Wiki 页面列表，选择页面后在右侧查看内容。

| 区域     | 说明                                        |
| -------- | ------------------------------------------- |
| 页面树   | 按类型、标题和更新时间浏览页面              |
| 页面正文 | 渲染 Markdown、frontmatter、引用和 wikilink |
| 来源     | 展示支撑当前页面的 Source Document          |
| 媒体     | 展示图片、caption、来源页和预览             |
| 版本     | 查看当前页面的历史版本和 diff               |

如果页面内容看起来不符合预期，先检查来源引用，再检查任务时间线中的分析和生成事件。

## 步骤 2：使用 Graph View

Graph View 用于理解页面之间的关系。它应占据主要可视区域，支持选择节点、下钻、上钻和查看详情。

| 操作         | 结果                             |
| ------------ | -------------------------------- |
| 点击节点     | 右侧显示页面详情、来源和关系     |
| 点击边       | 显示关系类型、权重和证据         |
| 点击图谱洞察 | 高亮相关节点或边，并显示解释     |
| 下钻         | 查看某个社群、主题或实体周边关系 |
| 上钻         | 返回更高层级的知识库视图         |

图谱关系来自 wikilink、shared source、common neighbor、type affinity、生成证据关系和社群检测，不应只依赖向量相似度。

## 步骤 3：查看 Graph Insights

图谱洞察用于发现结构性问题和机会。

| 洞察             | 说明                     |
| ---------------- | ------------------------ |
| isolated pages   | 页面缺少足够关系         |
| bridge pages     | 页面连接多个主题区       |
| communities      | 高度相关页面群组         |
| unexpected links | 可能值得确认的跨主题关系 |
| source clusters  | 多个页面共享相同资料来源 |
| missing links    | 可能应该建立但缺失的关系 |

点击洞察卡片时，图谱视图应同步变化，帮助管理员看到它具体指向哪些节点和关系。

如果点击洞察后图谱没有变化，需要检查洞察卡片是否带有关联节点 ID、边 ID 或 community ID。洞察只是说明文字是不够的，必须能把用户带回可视化图谱中的具体位置。

## 步骤 4：处理 Knowledge Check

Knowledge Check 是质量提示。常见处理方式：

| 类型     | 建议动作                       |
| -------- | ------------------------------ |
| 缺源     | 上传更多资料或修正页面来源     |
| 断链     | 补充目标页面或调整 schema      |
| 重复候选 | 检查是否需要合并页面或添加别名 |
| 冲突候选 | 查看来源证据，决定是否补充说明 |
| 弱检索   | 调整标题、摘要、标签或检索配置 |

处理后继续通过版本和检索调试验证影响。

## 步骤 5：查看版本和 Diff

版本页展示知识库级和页面级变化。

| 对象              | 用途                                  |
| ----------------- | ------------------------------------- |
| Knowledge Version | 表示知识库整体状态                    |
| Change Set        | 表示一次操作产生的变化集合            |
| Page Version      | 表示单个页面的历史                    |
| Diff              | 比较前后 Markdown、frontmatter 和来源 |
| Rollback          | 回到某个可恢复状态                    |

回滚会形成新的记录，历史记录继续保留。

建议每次生产批量入库后做一次抽查：

| 检查项          | 通过标准                           |
| --------------- | ---------------------------------- |
| Wiki 页面       | 页面标题、摘要、正文和来源引用存在 |
| Graph View      | 主要页面之间有可解释关系           |
| Graph Insights  | 洞察点击后能定位到节点或边         |
| Knowledge Check | findings 带有证据和可操作说明      |
| Versions        | 能看到本次入库产生的 Change Set    |
| Retrieve        | 能命中新增或更新页面               |

## 生产注意事项

- 如果 Wiki 页面缺失，先看关联任务是否 completed。
- 如果图谱为空，检查页面关系、Graph index status 和任务元数据。
- 如果检索命中差，先看页面标题、摘要、引用和 context budget，再看模型。
- 删除资料后，相关页面和图谱应通过异步清理或后续版本体现变化。
