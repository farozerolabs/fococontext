# 知识库流程

## 引言

本页说明如何从后台创建和维护 Knowledge Base。OpenAPI 创建方式见 OpenAPI 说明文档，本页侧重后台操作和配置含义。

## 步骤 1：创建知识库

在主页点击“新建知识库”，填写基础信息：

| 字段     | 说明               | 建议                           |
| -------- | ------------------ | ------------------------------ |
| 名称     | 管理后台显示名     | 用业务或数据集名称             |
| slug     | 稳定短标识         | 仅使用小写字母、数字和短横线   |
| purpose  | 知识库目标         | 写清受众、用途、领域和输出风格 |
| 输出语言 | Wiki 页面生成语言  | 与资料语言或最终用户语言一致   |
| schema   | 页面类型和字段规则 | 第一次使用默认值               |

保存成功后，系统返回 Knowledge Base ID。复制这个 ID，用于上传资料、Retrieve、Webhook 和自动化脚本。

## 步骤 2：检查概览

知识库概览用于回答三个问题：

1. 当前版本是否可用于检索。
2. 最近哪些资料和任务改变了它。
3. 哪些页面、图谱洞察、质量发现和版本需要关注。

| 指标               | 含义                          |
| ------------------ | ----------------------------- |
| retrieve readiness | 当前是否可以被 Retrieve 使用  |
| latest version     | 最新知识版本                  |
| source count       | 已关联资料数量                |
| job status         | 最近入库任务状态              |
| graph status       | 图索引和洞察是否可用          |
| check findings     | 需要关注的 Knowledge Check 项 |

## 步骤 3：配置业务规则

进入知识库设置，按需修改：

| 配置              | 什么时候改                         |
| ----------------- | ---------------------------------- |
| purpose           | 生成结果偏离业务目标时             |
| schema            | 页面类型、字段或命名规则需要调整时 |
| output language   | 需要切换 Wiki 输出语言时           |
| retrieval budget  | Retrieve 返回内容过长或过短时      |
| graph expansion   | 需要更多或更少图谱扩展时           |
| Markdown contract | 需要导出到外部 Wiki 或代码仓库时   |

大量入库正在运行时，避免频繁改变 schema 和输出语言。建议等当前任务完成后再调整。

## 步骤 4：导入资料

可以从资料页上传文件，也可以使用 OpenAPI 或 Source Watch。所有资料最终都会创建或关联 Ingest Job。

推荐第一次使用顺序：

1. 上传 1 个小文件。
2. 等任务完成。
3. 检查 Wiki 页面和 Graph View。
4. 在检索调试页验证 Retrieve。
5. 再批量上传或启用 Source Watch。

## 步骤 5：查看版本

每次入库、删除、回滚或外部提交都会形成版本记录。版本页可以查看：

| 内容              | 用途                 |
| ----------------- | -------------------- |
| Knowledge Version | 知识库整体版本       |
| Change Set        | 本次改变的页面和来源 |
| Page Version      | 单个页面的历史版本   |
| Diff              | 变更前后差异         |
| Rollback Record   | 回滚操作和结果       |

版本保留审计能力，但不要求管理员手动批准每次入库。

## 步骤 6：删除知识库

删除知识库是危险操作。后台会立即从普通列表隐藏它，并阻止继续操作。实际清理会异步执行，包括数据库记录、S3 对象、索引和相关任务状态。

删除前建议：

- 导出需要保留的 Markdown。
- 确认没有生产服务仍在调用该 Knowledge Base ID。
- 查看是否有正在运行的任务。
- 确认 Webhook 接入方能处理删除或不可用错误。

## 常见问题

| 问题               | 处理方式                                             |
| ------------------ | ---------------------------------------------------- |
| slug 无效          | 使用小写字母、数字和短横线                           |
| 保存后列表没更新   | 刷新主页或检查 API 错误 toast                        |
| 知识库不可检索     | 确认至少一个任务 completed 且索引 ready              |
| 删除后旧页面还能点 | 后台会二次校验资源状态并跳出旧详情                   |
| API 返回 not found | 资源可能已删除或清理中，检查 `request_id` 和服务日志 |
